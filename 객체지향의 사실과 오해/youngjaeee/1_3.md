## I. 협력하는 객체들의 공동체

### 1. 객체지향의 목표: 실세계의 모방(철학적 관점) vs. 새로운 세계를 창조(실무적 관점)
### 2. 실세계에서 객체는 역할과 책임을 부여받고, 서로 협력하여 사회를 이룸.
		· 역할: 책임 개념을 내포. 각 객체의 고유한 성질이다.
			- 여러 객체가 동일한 역할 수행 가능
			- 역할은 대체 가능함
			- 각 객체는 책임 수행 방법을 자율적으로 택함
			- 하나의 객체가 여러 역할 동시 수행 가능
			- 각 객체에 적절한 책임을 부여해야 효율적인 프로그램 설계 가능
		· 협력: 요청과 응답이 서로 반대 방향으로 이루어짐.
			- 객체는 요청을 충분히 수용하고 요청할 수 있어야 함(협력적)
				└ 메시지 전송 객체는 송신자, 수신 객체는 수신자
			- 객체는 요청을 행하는 방법을 스스로 결정함(자율적) → 캡슐화와 관련됨!
				└ 처리하는 방법은 메서드(함수 or 프로시저)
### 3. 객체지향 개념 정리:
		· 시스템을 자율적 객체의 공동체로 간주하고 분할
		· 자율적 객체는 상태(state)와 행위(behavior) 지니며 역할(책임)이 있음
		· 객체는 목표 달성을 위해 다른 객체와 협력함. 역할은 책임의 집합
		· 객체는 협력을 위해 메시지 주고 받으며, 메서드 통해 문제 해결함.
### 4. 객체지향은 클래스의 구조와 메서드가 아닌 객체의 역할, 책임, 협력에 집중해야 함!


## II. 이상한 나라의 객체

### 1. 객체: 하나의 개별적인 실체로 식별 가능한 물리적 or 개념적인 사물
		· 상태: 과거의 행동 이력과 현재 행동의 결과 예측을 위해 저장하는 값
			- 프로퍼티: 상태를 구성하는 모든 특징, 정적임.
				└ 단순한 값(attribute)와 링크(다른 객체 참조)로 이루어짐.
			- 프로퍼티 값: 시간에 따라 변경되며, 동적임.
			- 캡술화 통해 행동만 노출, 상태는 숨김으로써 자율성과 안정성 높임.
		· 행동: 객체의 상태 변경시키는 것, 부수 효과를 초래함.
			- 객체의 행동은 상태에 영향을 받고 상태를 변경시킬 수도 있음
			- 상호작용이 현재 상태에 어떻게 의존하고 어떻게 변경시키는지
			- 외부의 요청과 메시지에 응답하기 위한 것으로 외부에 가시적이어야 함.
		· 식별자: 객체를 서로 구별할 수 있게 하는 특정한 프로퍼티
			- 값: 식별자가 없으며 불변상태. 상태를 통한 동등성 검사로 비교
				└ 값 객체(value object)
			- 객체: 식별자가 있으며 상태 변경 가능. 식별자 통한 동등성 검사로 비교				└ 참조 객체(reference object), 엔티티(entity)
### 2. 객체의 성질
		· 상태를 가지며 변경 가능함.
		· 행동으로 상태 변경시킴. 결과는 상태에 의존적이며 행동 순서가 결과에 영향 미침.
		· 객체는 상태와 상관없이 식별자 통해 유일하게 식별 가능
### 3. 객체 부연설명
		· 상태 조회는 쿼리(query), 상태 변경 작업을 명령(command)이라고 함.
		· 객체 접근 위해서는 객체가 제공하는 쿼리와 명령 인터페이스로만 가능 → 캡슐화
		· 객체 설계를 위해서는 상태 보다는 행동 우선적으로 고려해야 할 필요가 있음.
				└ 캡슐화, 응집도 향상시키고, 재사용성 뛰어난 객체 생성을 위함.
		· 객체지향은 실세계를 은유적으로 표현, 의인화로 현실 객체보다 더 많은 역할 수행.

## III. 타입과 추상화

### 1. 추상화: 특정 절차, 물체를 생략하여 양상과 구조를 명확하게 이해하고 복잡도 낮추는 방법
		· 구체적 사물 간 공통점만 취하고 차이점을 버리는 일반화로 단순하게 만듦.
				└ 개념(concept)을 통해 공통점 기반으로 객체 그룹화
				└ 객체는 특정 개념을 적용 가능한 구체적인 사물이고,
				└ 개념을 적용한 객체를 개념의 인스턴스라고 한다.
		· 중요한 부분 강조 위해 불필요한 세부 사항 제거하여 단순하게 만듦.
### 2. 개념: 특정한 객체가 어떤 그룹에 속할지 결정함.
		· 심볼(symbol): 개념을 가리키는 명칭
		· 내면(intension): 개념의 완전한 정의, 객체가 개념에 속하는지 여부에 확인
		· 외연(extension): 개념에 속하는 모든 객체 집합
### 3. 분류: 객체에 특정한 개념을 적용하여 어떤 집합의 멤버로 나누는 것
		· 분류는 개념을 적용하여 객체의 복잡성을 줄이기 위한 추상화 도구이다!
### 4. 타입: 개념과 유사한 의미 (컴퓨터 내부적인 측면에서)
		· 데이터가 어떻게 사용되느냐에 관한 것임.
		· 타입에 속한 데이터의 메모리 저장 방법은 철저히 숨김
		· 객체의 타입 소속은 수행하는 행동을 통해 분류되며, 내부적 표현은 철저히 감춰짐.
				└ 같은 타입의 객체는 서로 다른 데이터 저장 가능(다형성)
				└ 외부에서는 객체의 행동만 확인이 가능함(캡슐화)
### 5. 객체의 상/하위 개념: 타입 간의 포함 관계를 내포
		· 일반화 → 좀 더 일반적인 타입: 슈퍼타입
		· 특수화 → 좀 더 특수한 타입: 서브타입
				└ 각 객체가 하는 행동을 통해 일반화, 특수화 관계를 결정한다
### 6. 타입을 통한 추상화: 객체의 동적인 특성을 추상화하여 상태 변경이라는 복잡성 단순화
### 7. 동적 모델과 정적 모델
		· 동적 모델: 객체가 어떻게 변하고 행동하는 지 포착하는 것
				└ 스냅샷(특정 시점), 객체 다이어그램
		· 정적 모델: 객체가 속한 타입의 정적인 모습
				└ 타입 모델(시간에 독립적)
